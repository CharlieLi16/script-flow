# 翻页分镜动画提词模板

Prompt 不写在后端源码里。把下面模板复制到 **提词库**，出图时选择「翻页动画」模式并套用该模板；前端会替换占位符后，把最终文本原样发给模型。

## 占位符

| 占位符 | 含义 |
|--------|------|
| `{frameCount}` | 总帧数：4 / 8 / 16 |
| `{columns}` | 列数（默认布局：4→2，8→4，16→4） |
| `{rows}` | 行数（默认布局：4→2，8→2，16→4） |
| `{userPrompt}` | 你在 Prompt 输入框里写的动画内容 |

## 推荐模板

```text
请生成一张用于逐帧动画的连续分镜表，共包含严格且仅包含 {frameCount} 个画面，排列为 {columns} 列 × {rows} 行。

动画内容：{userPrompt}

要求：
1. 所有格子必须尺寸完全一致，按从左到右、从上到下的顺序表现一个连续、平滑、时间间隔均匀的动作过程。
2. 每格必须保持同一角色、物体、场景、构图、镜头位置、焦距、光线、色彩和画风；只改变完成动作所必需的部分。
3. 第一格是动作起点，最后一格是动作终点，中间各格自然过渡；如果要求循环动画，最后一格必须能平滑衔接第一格。
4. 输出必须是一张无缝帧图集（seamless frame atlas），格子边界只是不显示出来的裁切坐标。相邻画面必须像素贴像素直接相接，每个画面铺满自己的整个区域直到裁切边缘。
5. 绝对不要画出任何可见分界：禁止白线、黑线、彩色线、间距、留白、边框、描边、分栏线、十字线、圆角或阴影；特别禁止在行列交界处出现横线或竖线。
6. 不要使用漫画分格、故事板面板、相册拼贴或联系表的视觉样式。整张图上只能看到连续铺满的画面内容，程序会按不可见的等分坐标直接裁切。
7. 不得合并格子、重复格子、遗漏格子或在单个格子内再画小分镜。
8. 每个格子的主体比例和位置必须稳定，背景不得跳动，画面必须铺满格子边缘，便于程序按固定网格精确裁切。
9. 若提供参考图，严格保持参考图中的角色身份、服装、物体造型、场景和视觉风格。

最终只输出完整分镜表图片，不要输出解释或额外内容。
```

## 使用步骤

1. 打开提词库 → 新建 → 粘贴上面模板 → 保存（例如命名为「翻页分镜表」）。
2. 选择时间线节点 → AI 出图切换到 **翻页动画**。
3. 选择帧数（4 / 8 / 16）和 FPS（默认 4）。
4. 点「选用模板」，选中刚才保存的提词。
5. 在 Prompt 里只写动作内容，例如：`书页从右向左翻开，最后一页停在 Believing is seeing`。
6. 确认下方「最终 Prompt」预览无误后生成；后端会按网格裁切并按序播放。

## 默认网格

前端默认映射（可改 UI，后端只认请求里的 `columns` / `rows`）：

- 4 帧 → `2 × 2`
- 8 帧 → `4 × 2`
- 16 帧 → `4 × 4`

修改提词库模板后，下一次生成会立即使用新文案，无需改代码。

## 文字清晰度

16 帧分镜表里，每格大约只有整图的 1/16。若整图约 1024×1024，单格约 256×256，细小文字几乎必然发糊。

建议：

1. **有字优先 4 帧**（2×2），单格约 512×512，可读性好很多。
2. **尺寸选 1:1**，让正方形网格均分像素。
3. Prompt 里要求：字少、字号大、粗笔画、高对比、尽量占画面主要区域。
4. 系统会在裁切后把过小的帧放大到约 768 边长并轻微锐化，能改善预览观感，但**无法凭空补回模型没画清的笔画**。
5. 若必须 16 帧且字要清晰，更稳的路线是分两张 8 帧表，或关键帧单独出图再合成。
